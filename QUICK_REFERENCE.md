# Git Workflow & Version Management Summary

## 🎯 Quick Start

This project uses **Conventional Commits** + **Semantic Versioning** + **Automated Releases**.

### Initial Setup

```bash
# Clone and setup
git clone <your-repo>
cd node-microservices-ecommerce
npm run setup

# That's it! Husky hooks are now active.
```

### Daily Development

```bash
# 1. Start from main
git checkout main
git pull origin main

# 2. Create feature branch
git checkout -b feature/awesome-feature

# 3. Make changes and commit with conventional message
git add .
git commit -m "feat(products): add product search by name"

# 4. Push and create PR
git push origin feature/awesome-feature
# → Create PR on GitHub
```

### Making a Release

```bash
# 1. Merge all PRs into main
git checkout main
git pull origin main

# 2. See what will be released
npx standard-version --dry-run

# 3. Create release
npm run release:minor   # for features
# OR
npm run release:patch   # for bug fixes only
# OR
npm run release:major   # for breaking changes

# 4. Push (this creates tag and GitHub release)
git push --follow-tags
```

## 📋 Commit Message Cheat Sheet

```
type(scope): short description

feat(products): add category filtering
fix(orders): prevent duplicate orders
docs: update installation guide
refactor(user): simplify login logic
test(order-service): add unit tests
chore: update dependencies to latest
feat(api-gateway)!: remove legacy auth endpoint

BREAKING CHANGE: Old authentication endpoint removed.
Migration required within 30 days.
```

**Types**: `feat` (minor) | `fix` (patch) | `docs` | `style` | `refactor` | `perf` | `test` | `chore` | `revert` | `build` | `ci`

**Scope (optional but recommended)**:
- `user-service`, `product-service`, `order-service`
- `api-gateway`, `event-bus`, `frontend`
- `shared`, `docker`, `ci`

**Breaking change**: Add `!` after type, include `BREAKING CHANGE:` in body

## 🏷️ Version Numbers

Format: `MAJOR.MINOR.PATCH` (e.g., `1.2.3`)

| Change | Type | Version Bump |
|--------|------|--------------|
| New feature | `feat` | MINOR (1.2.0 → 1.3.0) |
| Bug fix | `fix` | PATCH (1.2.3 → 1.2.4) |
| Breaking change | `feat!` or `fix!` | MAJOR (1.2.3 → 2.0.0) |
| Docs/chore/etc | none | no bump |

## 🆘 Troubleshooting

### "Commit message does not conform to conventional commits"
→ Fix message: `git commit --amend -m "fix(service): proper format"`

### "No commits found since last release"
→ Probably no conventional commits. Check `git log`. Use proper types.

### Need to edit CHANGELOG manually?
Edit `CHANGELOG.md` under `## [Unreleased]` section, then:
```bash
git add CHANGELOG.md
git commit -m "docs: update changelog with manual entries"
```

### Undo a release
```bash
# Delete tag
git tag -d v1.2.0
git push origin :v1.2.0

# Reset to previous commit
git reset --hard HEAD~1

# Re-release correctly
npm run release:patch
```
